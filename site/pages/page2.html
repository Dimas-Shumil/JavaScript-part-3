2. Циклы

Циклы помогают избавиться от многократного написания однотипного кода. Они могут выполнить какую-нибудь часть кода несколько раз. Например, вывести 100 раз на экран слово «Привет».

Вместо того, чтобы 100 раз писать команду вывода «Привет», мы создадим цикл, который вызовет эту команду 100 раз.

Каждый проход цикла называется итерацией. Проход цикла или итерация — это повторение записанных в цикле команд.

Чтобы лучше понять, что такое проход цикла или итерация, выберите одну из панд, которые катаются с горки на картинке в начале юнита. Каждый раз когда панда начинает забираться по лестнице — начинается новая итерация.

Ещё один пример. Нам нужно вывести для пользователя уведомление через определённое количество секунд. Для решения этой задачи мы можем написать таймер. Пользователь вводит время, когда таймер должен сработать, нажимает кнопку «Старт». Таймер начинает работу.

Чтобы не писать код под каждое возможное время, мы можем получить от пользователя значение и в цикле сверять текущее время с тем, что задал пользователь.

В верхнем ромбе мы создаём условие, которое должно иметь булевый тип.

Например:

1 === 2 // false

'some string' === 'some string' // true

45 > 46 // false

Если условие истинно, то есть имеет значение true, то мы движемся по левому пути к блоку «Что-то делаем с переменной».

В этом блоке может содержаться любой код, который необходим разработчику. Как только блок кода выполнится, мы опять попадём в ромб с условием. Этот блок называется «Тело цикла».

А что, если условие станет ложным? Например, значение переменной будет больше необходимого значения. Тогда мы пойдём по правому пути — к блоку «Вывод сообщения».

Код внутри тела цикла не будет выполнен, пока не выполнится условие.

Разновидности циклов

В JavaScript есть несколько видов циклов:

➤ while

➤ do…while

➤ for

Разберём каждый из них подробнее.

while

Цикл while сначала проверяет, что условие истинно — имеет значение true. А уже потом начинает выполнение тела цикла. Другими словами: сначала подумает, а потом сделает.

У этого цикла следующий синтаксис:

while( условие ) {

// тело цикла

}

let value = 0; // создаём изменяемую переменную

while (value ‹ 10) { // проверяем условие

value = value + 1; // увеличиваем переменную на 1

}

Если нам нужно пропустить какую-нибудь итерацию цикла, например value === 5, то для этого у нас есть 2 специальных команды.

continue

Цикл сразу переходит на следующую итерацию. Код, который находится после данной команды, не запускается.

В итоге выполнения этого кода мы получим вывод в консоль: 1, 2, 3, 4, 6, 7, 8, 9. Обратите внимание, что цифры 5 в выводе нет.

В примере используется метод console.log() . Он выводит значение в консоль разработчика, которую можно открыть в браузере. Например, в Chrome:

1. нажимаем на правую кнопку мыши в окне браузера,

2. выбираем «Просмотреть код»,

3. в появившейся панели открываем вкладку «Консоль».,

for

Этот цикл сложнее предыдущих циклов по устройству. Но при этом он самый распространённый.

for состоит из трёх частей:

1. Начальное значение — задаётся один раз в начале цикла.

2. Условие — проверяется на каждой итерации цикла.

3. Шаг — выражение выполняется после каждой итерации.

Записывается цикл следующим образом:

for(начальное значение; условие; шаг цикла) {

// тело цикла

}

Обратите внимание! Каждая из частей цикла отделяется от следующей точкой с запятой.

Ранее мы говорили, что переменным нужно давать осмысленные, говорящие названия.

Цикл for не исключение. Однако чаще всего в нём используют название начального значения, которое состоит из одной буквы: i, k, j.

Например, для вывода цифр от 0 до 10 цикл for будет выглядеть так:

for (let i = 0; i ‹= 10; i = + 1) {

console.log(i);

}

Как работает этот код:

1. Мы задали начальное значение переменной i в первой части.

2. Во второй части проверяем, меньше числа 10 или равна ему переменная i.

3. В третьей части цикла увеличиваем значение переменной на 1 после каждой итерации

4. Как только значение переменной станет больше 10, цикл перестанет выполняться.

Главная особенность цикла for: все части в нём — необязательные.

let i = 0;

for (; i ‹ 10; i += 1) {

// тело цикла

}

Также вы можете не объявлять центральную часть цикла — условие:

for (let i = 0; ; i += 1) {

// тело цикла

}

И даже не объявить третью часть цикла — шаг:

for (let i = 0; i > 10;) {

// тело цикла

}

Важно! Даже если какая-то часть цикла пропускается, точку с запятой всё равно нужно поставить.

Для цикла for, также как и для цикла while, работают команды continue и break:

for (let i = 0; i ‹ 10; i += 1) {

if (i === 5) { // если i равно 5, то
continue; // пропускаем вызов следующего кода и переходим к следующей итерации цикла
}
console,log(i); // 0, 1, 2, 3, 4, 6, 7, 8, 9

}

for (let i = 0; i ‹ 10; i += 1) {

if (i > 5) { // если i больше 5, то
break; // прерываем выполнение цикла
}
console.log(i); // 0, 1, 2, 3, 4, 5

}

Использование циклов на практике

for — самый используемый цикл. Поиск по Github выдаёт 28 миллионов файлов, где встречается данная конструкция. Для цикла while и do…while эта цифра равна 14.2 миллиона.

Он используется для:

➤ перебора элементов массива;

➤ заполнения массива значениями;

➤ перебора элементов массивоподобных объектов.

Циклы while и do…while используются, когда изначально неизвестно количество необходимых повторений.

Например, цикл do…while может быть настроен на ввод пользователем чисел. Мы не знаем, сколько чисел захочет ввести пользователь. Но на каждое мы должны делать какое-то действие. Например, подсчитывать сумму всех введённых чисел.

Давайте разберём эту задачу подробнее.

let num; // здесь будем сохранять число введённое пользователем

let sum = 0; // переменная для хранения суммы чисел

// используем цикл do...while так как нужно выполнить код хотя бы один раз

do {

num = +prompt('Введите число'); // получаем значение от пользователя и приводим к типу number
if (typeof num === 'number') { // проверяем, что в переменной записано число, так как может быть значение NaN или null, если пользователь отказался вводить данные
console.log(`Введено число: ${num}`); // вывод в консоль
sum += num; // прибавляем к переменной введённое значение
console.log(`Сумма введённых чисел: ${sum}`); // вывод в консоль
}

} while (num) // будет выполняться, если в переменной записано значение, отличное от null, false или undefined

console.log(`Общая сумма: ${sum}`); // вывод в консоль

Часто можно встретить вложенность одного цикла for в другой.

Сейчас эта возможность поможет нам вывести таблицу умножения в консоль. А далее при работе с массивами и объектами сможем работать с вложенными в них элементами.

Пример с выводом таблицы умножения:

for (let i = 1; i ‹= 10; i += 1) {

for (j = 1; j ‹= 10; j += 1) {
console.log(`${i} * ${j} = ${i * j}`);
}
console.log('\n');

}

В результате мы получим вывод:

1 * 1 = 1

1 * 2 = 2

1 * 3 = 3

1 * 4 = 4

1 * 5 = 5

1 * 6 = 6

1 * 7 = 7

1 * 8 = 8

1 * 9 = 9

1 * 10 = 10

Давайте разберём, как это работает:

1. for (let i = 1; i ‹= 10; i += 1) { — создаём внешний цикл.

➤ Создаётся переменная i со значением 1, добавляем условие по достижении которого цикл будет завершён i ‹= 10 и указываем шаг цикла равный 1.

➤ Внешний цикл начинает свою работу.

1. Получает значение переменной i.

2. Проверяет, что переменная соответствует условию.

3. Если переменная соответствует условию, то запускается код внутри цикла.

4. Ждёт выполнения внутреннего кода.

5. Изменяет значение переменной i.

6. Начинает работу с п. 1.

2. for (j = 1; j ‹= 10; j += 1) { — создаём внутренний цикл.

➤ Создаётся переменная j со значением 1, добавляем условие по достижении которого цикл будет завершён j ‹= 10 и указываем шаг цикла равный 1.

➤ Внутренний цикл начинает работу

1. Получает значение переменной j.
2. Проверяет, что переменная соответствует условию.
3. Если переменная соответствует условию, то запускает код внутри цикла.
4. Ждёт выполнение внутреннего кода.
5. Изменяет значение переменной j.
6. Начинает работу с п. 1.

3. С помощью шаблонной строки выводится сообщение в консоль из внутреннего цикла.

4. После завершения внутреннего цикла выводится сообщение в консоль.

Таким образом, внешний цикл запускается всего 10 раз, а внутренний каждый раз после запуска внешнего — 100 раз.

Бесконечные циклы

В процессе написания кода можно случайно создать бесконечный цикл.

Иногда разработчики создают их специально. Например, ожидание действий пользователя можно сделать как бесконечный цикл. Представим, что две программы выполняются параллельно. Одну из них пользователь использует в текущий момент, а вторая ждёт определённого действия от пользователя, чтобы начать выполнение. Дождавшись его, вторая программа начнёт выполнять свой код. Она может после этого как завершиться, так и опять встать в режим ожидания.

Но это скорее исключение из правила.

Бесконечные циклы могут быть опасны.

От бесконечного цикла среда, в которой он исполняется, может зависнуть. В лучшем случае вы сможете закрыть среду выполнения (например, окно браузера), и находящийся в ней код перестанет работать. В худшем случае компьютер придётся экстренно перезагружать, потому что система перестанет отвечать на действия пользователя.

Вот пример бесконечного цикла:

let i = 0;

while (i ‹ 10) {

console.log(i);

}

В примере происходит инициализация переменной и запуск цикла. Значение переменной нигде не изменяется и проверка i ‹ 10 всегда будет выдавать true. Цикл вышел бесконечным.
