3. Массивы

Мы с вами изучили примитивные типы данных — string, number, boolean, null и undefined. Теперь пора изучить структуры, которые могут хранить эти данные.

Первая такая структура — массив.

Массив — это структура, в которой можно хранить коллекции элементов. Например, числа, строки, другие массивы и т.д.

Элементы хранятся в том порядке, в котором они были добавлены.

Один из способов создания массива — квадратные скобки [].

const array = []; // создали пустой массив

Также можно создавать массив с помощью конструктора new Array():

const array = new Array();

Массив можно создать сразу с элементами. Элементы в массиве разделяются запятыми.

const months =

['январь','февраль','март','апрель','май','июнь','июль','август','сентябрь','октябрь','ноябрь','декабрь'];

Элементов может быть много и они могут быть какими угодно. Максимальное число элементов — 4.294.967.296 (Array.length — материал из документации MDN).

const different = ['string', 2, 43, false, null, [1, 3, 'inner array']];

Прежде, чем начать работу с массивами и их методами, давайте разберёмся, как они хранятся в памяти компьютера.

Хранение данных

В JavaScript существует 2 способа хранения данных: по ссылке и по значению.

По значению хранятся примитивные типы: строки, числа, булевый тип и т.д.

Это означает, что при каждом создании переменной выделяется место в памяти, куда записывается значение переменной.

Сложные типы данных хранятся по ссылке. Это значит, что в памяти выделяется небольшая область. И в переменную записывается ссылка на эту область.

Способы хранения можно представить как разговор двух людей.

Примитивный тип данных: нам говорят фразу: «Наступило лето». Это фраза, которую мы услышали и сразу поняли. Нам не нужно куда-то лезть и узнавать её значение.

Ссылочный тип данных: нам говорят «Возьми книгу Л.Н. Толстого «Война и мир», открой 35-ую страницу и прочитай 10-ую строчку сверху». Здесь содержание строки нам сначала непонятно. Как только мы её найдём, мы узнаем, что там написано. То есть нам дали «ссылку» на нужное значение.

У ссылочного типа есть ещё одна особенность — изменение значения. Давайте разберём эту особенность на примере.

Допустим, мы создаём переменную А, присваиваем ей значение. Затем решили создать переменную Б и присвоить ей значение переменной А. Дальше мы решили поменять значение в переменной А, зная, что у нас есть переменная Б, в которой хранятся старые данные.

Меняем значение переменной А и обнаруживаем, что значение в переменной Б тоже изменилось.

Это произошло из-за того, что в переменной А хранится не определённое значение, а лишь ссылка на него. В переменной Б значение меняется потому, что она «следит» за той же ссылкой, что и переменная А.

Также внутренние значения в таких переменных можно изменять, даже если переменная была объявлена с помощью ключевого слова const.

Изменение значения примитивного типа:

let primitiveA = 5;

let primitiveB = primitiveA;

primitiveA = 10;

console.log(primitiveA); // 10

console.log(primitiveB); // 5

Изменение значения ссылочного типа:

const linkTypeA = ['first', 'second', 'third'];

const linkTypeB = linkTypeA;

linkTypeA[3] = 'fourth'; //добавление элемента в массив

console.log(linkTypeA); // ['first', 'second', 'third', 'fourth']

console.log(linkTypeB); // ['first', 'second', 'third', 'fourth']

Теперь мы знаем, как хранятся массивы в памяти и можем начать их изучать.

Получение элемента массива

Нумерация элементов в массиве начинается с нуля. Чтобы получить из массива с месяцами «март», мы должны написать:

months[2] // март

А для получения первого месяца в году:

months[0] // январь

За один запрос можно получить только один элемент.

month[0] ✅

month[0, 2] ❌

Не каждый пользователь знает про эту особенность. Поэтому в коде нужно либо прибавлять к полученному от пользователя значению 1, либо удалять.

Номер элемента в массиве называется индексом. В предыдущих примерах мы получали элементы с индексами 0 и 2.

Методы и свойства массива

У массива есть методы и свойства, которые облегчают работу с ним. Есть свойство для получения длины массива, методы удаления или добавления элемента, а также методы для проверки нахождения элемента в массиве.

Теперь перейдём к изучению методов и свойств массивов.

length

Свойство называется length. Оно показывает нам длину массива:

months.length // 12

Обратите внимание, что свойство length считает длину с 1. А первый индекс массива – 0.

С помощью свойства length легко получить последний элемент массива, даже не зная его длины.

months[months.length - 1] // Декабрь

В квадратных скобках мы обращаемся к свойству length и отнимаем 1. В массиве находится 12 элементов и свойство length показывает нам их количество. Индекс первого элемента равен нулю. Значит, при 12 элементах индекс последнего элемента будет равен 11.

Такой приём используется в циклах, когда нужно пробежаться по всей длине массива.

for (let i = 0; i months.length; i += 1) {

console.log(months[i]) // в консоль выводятся все элементы массива

}

При обращении к несуществующему индексу массива, мы получаем undefined.

months[12] // undefined

Методы добавления элементов

У массивов есть начало — индекс 0. И конец — индекс равный длине массива -1

Для массивов в JavaScript были сделаны методы добавления элементов как в начало, так и в конец.

Добавление в начало массива считается «дорогой» операцией с точки зрения производительности. В этом случае JavaScript будет вынужден увеличить индексы всех элементов массива на 1 и сдвинуть их на одну позицию.

Иначе обстоит дело с добавлением в конец массива. JavaScript уже знает длину массива и после последнего элемента добавляет ещё один. Остальные элементы массива при этом не затрагиваются.

Давайте познакомимся с этими методами.

push

Метод добавляет элемент в конец массива и возвращает новую длину массива.

Это означает, что при вызове этого метода мы получаем значение новой длины массива и можем записать его в переменную.

const arr = ['First', 'Second'];

const newArrLength = arr.push('Third');

console.log(newArrLength); // 3

console.log(arr); // ['First', 'Second', 'Third']

Этот метод — «дешёвый» для использования. Потому что массив не пересчитывается, а элемент добавляется в конец.

С помощью этого метода и цикла for мы можем заполнить массив большим количеством чисел:

const numbers = [];

for (let i = 1; i ‹= 9999; i += 1) {

numbers.push(i);

}

console.log(numbers);// [1, 2, 3, 4, 5 ... 9999]

unshift

Добавляет элемент в начало массива и возвращает новую длину массива.

const arr = ['First', 'Second'];

const newArrLength = arr.unshift('Third');

console.log(newArrLength); // 3

console.log(arr); // ['Third', 'First', 'Second’]

Этот метод «дороже» для исполнения. Системе нужно пересчитать все индексы и вставить элемент в начало массива.

На маленьких массивах это не заметно. Если массив состоит из миллиона элементов, то при выполнении такой операции система может зависнуть на несколько секунд или дольше.

Методы удаления элементов

Есть методы удаления элементов как с начала, так и с конца массива. Но, в отличии от методов добавления, они возвращают значение удалённого элемента, а не новое значение длины массива.

pop

Удаляет последний элемент массива и возвращает удалённый элемент.

const arr = ['First', 'Second'];

const el = arr.pop();

console.log(el); // Second

console.log(arr); // ['First']

Эта возможность может быть полезна для проверки удаляемого элемента или для проведения над ним каких-нибудь манипуляций.

shift

Удаляет первый элемент массива и возвращает удалённый элемент.

const arr = ['First', 'Second'];

const el = arr.shift();

console.log(el); // First

console.log(arr); // ['Second']

Перебор элементов массива

Возьмём уже известный нам массив с месяцами:

const months =

['январь','февраль','март','апрель','май','июнь','июль','август','сентябрь','октябрь','ноябрь','декабрь'];

Перед нами поставили задачу. В начало каждого элемента нужно добавить его порядковый номер, чтобы получилось «1 – январь», «2 – февраль» и т.д.

Задача понятна. Массив небольшой и мы решили, что будем обращаться к каждому элементу по его индексу и изменять значение.

months[0] = '1 - январь';

months[1] = '2 - февраль';

...

months[11] = '12 - декабрь';

Посмотрите на код. 12 раз у нас дублируется одна и та же операция. Скорее всего, это можно было сделать красивее. Как — расскажем дальше.

Метод № 1

Например, можно сделать код аккуратнее с помощью цикла for. Он поможет нам в обходе массива.

Мы зададим начальный индекс массива в первой части цикла. Затем добавим условие, чтобы не выйти за границы массива. И в третьей части цикла будем увеличивать индекс на единицу.

Внутри цикла мы будем получать значение массива для текущего индекса и менять его.

for (let index = 0; index ‹ months.length; index += 1) {

months[index] = `${index + 1} - ${months[index]}`;

}

months[index] = `${index + 1} - ${months[index]}`;

Другой путь — воспользоваться циклом for…of.

Это вариация цикла for для массивов. У него проще структура:

➠ не нужно создавать переменную и задавать ей значение;

➠ не нужно создавать условие;

➠ не нужно писать шаг цикла.

for (let month of months) {

bloconsole.log(month);

}

Цикл for…of можно применять в случаях, когда нам не важен индекс элемента. Как в примере с выводом в консоль названия месяца.

Для решения задачи с изменением названия элемента в массиве этот метод не подойдёт. Но мы можем схитрить и не менять исходный массив. Мы будем добавлять уже изменённые названия в новый массив.

const newMonths = [];

let index = 1;

for(let month of months) {

newMonths.push(`${index} - ${month}`);

}

Давайте разберём работу этого цикла:

1. for(let month of months) { — создаётся переменная month, в которую записывается значение текущего элемента массива.

2.выполняется код внутри фигурных скобок.

3. в ранее созданную переменную month записывается значение следующего элемента списка.

Цикл сам знает, какова длина массива. И у нас не получится выйти за его предел.

С помощью этих циклов мы можем отфильтровать значения массива по какому-либо условию и записать итоговый результат в новый массив:

const arr = [1, 2, 3, 4, 5]; // создаём или получаем массив с числами

const newArrayWithEvenNumbers = []; // создаём пустой массив

for (let number of arr) { // перебираем исходный массив

if (number % 2 === 0) { // проверяем каждое значение на условие
if (number % 2 === 0) { // проверяем каждое значение на условие
}

}}

console.log(newArrayWithEvenNumbers) // 2, 4
