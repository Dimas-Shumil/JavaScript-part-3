4. Объекты

Объекты — это сложная структура данных. В JavaScript объектом является почти всё. Например, если вы сделаете вызов метода typeof для массива, то в ответ получите тип «объект».

В объекте хранятся значения любых типов. Каждое такое значение записывается в уникальный ключ. Ключ должен быть уникальным в рамках одного объекта. Можно сказать, что ключ — имя переменной, а его значение или свойство — значение переменной.

Создание объекта

Объект можно создать двумя способами:

· const obj = {};

· const obj = new Object();

Они равнозначны. Но чаще пользуются первым, потому что он проще. Остановимся на нём.

Между фигурных скобок можно указывать свойства объекта. Они отделены друг от друга запятыми.

Свойство объекта имеет формат ключ: значение.

vconst obj = {

first: 'First',
second: 'Second'

}

Объект можно представить в виде папки на рабочем столе компьютера. Ключ — название папки, значение — содержимое папки.

Например, мы хотим положить школьные фотографии за 10 класс в папку с названием «Школа 10 класс». Получается, что мы обращаемся к объекту с ключом «Школа 10 класс». А внутрь будем складывать фотографии — значения для данного ключа.

Ключ может быть только строкового типа, а значение может иметь любой тип.

const obj = {

first: 'string',
second: '1',
third: ['string','in','array'],
fourth{
inner: 'object'
}

}

Но здесь не обошлось без нюансов:

1. Если ключ состоит из одного слова, то можно его писать без кавычек.

2. Если ключ состоит из двух и более слов, то нужно писать в кавычках.

const men = {

name: 'Ivan',
'eyes color': 'green'

}

Допустим, пользователь на сайте заполняет анкету. Мы получаем в переменную name имя пользователя. Далее его имя нужно вместе с другими данными передать на сервер в виде объекта user.

Ранее мы добавляли в объект различные значения напрямую. А как быть, если значение записано в переменную?

Нам достаточно написать эту переменную в качестве свойства ключа:

const name = 'Василий';

const userName = {

userName = 'name',

}

Если в объекте ключ имеет то же название, что и переменная, то запись сократится:

const name = 'Василий';

const user = {

name,

}

Это равносильно записи:

const name = 'Василий';

const user = {

name = name,

}

Доступ к свойствам

Для получения значения элемента массива мы обращались к нему по индексу:

const arr = ['some', 'string'];

arr[0] // some

У объектов нет индексов. Но у них есть ключи, которые в рамках объекта должны быть уникальными.

Чтобы получить значение свойства объекта, нужно обратиться к нему по ключу:

const obj = {

some: 'value',
data: '2022-01-12'

}

obj.some //value

obj['some'] /value

Обращение к ключу объекта происходит через точку. Но объекты не так просты, как массивы. Здесь нет единственно верного способа.

Обращение к ключу с помощью точки возможно только для ключей, которые состоят из одного слова. Если имя ключа состоит из нескольких слов, то мы будем обращаться к нему с помощью квадратных скобок:

const man = {

name: 'Ivan',
'eyes color':'green'

}

man['eyes color']//green

Динамические свойства

Давайте представим, что у нас есть массив строк, которые являются названиями ключей объекта:

const keyName = ['first','second',therd];

И есть объект:

const obj = {

first: 'some',
second: 'Value',
third: '1'

}

Мы хотим вывести в консоль все значения свойств объекта. Как использовать имеющиеся данные, чтобы решить эту задачу?

На помощь приходит возможность получения значения свойства из переменной. Возьмём цикл for…of для массивов. В цикле каждое значение массива будем использовать как ключ для свойства объекта:

for (let key of keyNames) { // начинаем перебор элементов массива

console.log(obj[key]); // вместо переменной key подставится её значение

}

После выполнения цикла мы увидим в консоли значения ключей массива — Some, Value и 1.

Добавление свойств

Мы создали какой-то объект, но обнаружили, что забыли добавить несколько свойств в него. В этом нет ничего страшного — мы можем добавить их даже после создания объекта.

Вот, как это можно сделать:

const obj = {};

obj.first = 'Первый способ';

obj['second'] = 'Второй способ';

console.log(obj);

// obj = {

// first: 'Первый способ',

// second: 'Второй способ'

// }

Это похоже на доступ к свойствам объекта. Если свойства в объекте нет, то оно будет создано. А если свойство уже есть, то у него изменится значение.

Удаление свойства

Удаление свойства происходит с помощью ключевого слова delete:

delete obj.first // true

Изменение свойств

Изменение значения свойств происходит так же, как и добавление:

const user = {

name: 'Витя',

}

userName = 'Дима';

console.log(user);//{name:'Дима'}

Обратите внимание: если вы попытаетесь изменить значение свойства и напишите несуществующее в объекте свойство, то оно создастся и никакой ошибки вы не увидите.

Если где-то дальше в коде вы обращаетесь к нужному свойству, то вы ожидаете увидеть в нём значение. Так как значение было ошибочно записано в другое свойство, вы можете получить ошибку либо нужный код не будет выполнен.

const user = {};

user.age = 'Вова';

if (user.name === 'Вова') { // сейчас код внутри блока не будет выполнен. свойства name нет в объекте user

// что-то делаем, если пользователя зовут Вова

}

Поэтому перед операцией изменения значения стоит проверить существование свойства в объекте, чтобы избежать возможных проблем далее при обращении к нужному свойству.

Проверка существования свойства

Проверить существование можно двумя способами:

1 Через оператор равенства == или оператор строгого равенства ===.

obj.name === undefined

Если результат будет true, то свойства в объекте не существует. Но что делать, когда значение свойства undefined?

В этом случае нужно использовать оператор in. Значение нужного ключа записывается в виде строки. Также можно указать имя переменной, в которое записано значение ключа:

const key = 'name';

console.log('name' in obj)

console.log(key in obj)

Перебор свойств

Для перебора свойств объекта мы можем использовать специальную разновидность цикла for — for…in.

Записывается этот цикл также, как и цикл for…of:

for (let key in obj) {

// тело цикла выполняется для каждого ключа объекта

}

const user = {

name: 'Luna',
age: '40',
hairColor: 'black',

}

for (let key in user) {

// ключи
console.log(key); // name, age, hairColor
// значение ключей
console.log(user[key]); // Luna, 40, black

}

Деструктуризация объектов

Так же, как и массивы, объекты можно деструктурировать. Идея этой операции здесь та же самая — вытащить значение и создать переменную в одну строку. Но реализация отличается.

В массивах деструктуризация выглядит следующим образом:

const array = [1, 2, 3];

const [first, , third] = array; // 1 и 3

Мы можем придумать любые названия переменным. Чтобы получить нужное значение, нам нужно, чтобы у созданной переменной был такой же порядковый номер в деструктурирующем массиве, как у значения в основном массиве.

В объектах нам важен не порядок, а ключ:

const obj = {

first: 'First',
second: 'Second',
third: 'Third',

}

const { first, third, second } = obj; // First, Third, Second

Обратите внимание, что при деструктуризации массива мы используем массив для объявления переменных. А при деструктуризации объекта — объект.

Представим, что у нас есть такой код:

const first = 'First variable';

const obj = {

first: 'First',
second: 'Second'

}

Если мы проведём деструктуризацию объекта, то получим ошибку. Так как переменная с именем first уже есть в коде.

const { first, second } = obj; // Ошибка, переменная first создана ранее.

Чтобы избежать подобной ошибки, мы можем изменить имя деструктурируемой переменной. Для изменения имени нужно использовать двоеточие:

const { oldName: newName } = someObject;

const { first: firstObjValue, second } = obj;

console.log(first); // First variable

console.log(firstObjectValue); // First

Объект как массив

У объектов есть ещё интересная особенность. Мы можем получить все ключи объекта и сделать из них массив. Можем получить все значения объекта и тоже сделать из них массив. А можем получить все ключи и значения и сделать из них массив массивов!

В этом нам поможет конструктор Object со своими методами Object.keys, Object.values и Object.entries:

const obj = {

first: 'First',
second: 'Second'

Object.keys(obj) // [first, second]

Object.values(obj) // ['First', 'Second']

Object.entries(obj) // [[first, 'First'], [second, 'Second']]

}

Зачем нам может понадобиться массив массивов?

Например для вывода значений в таблицу.

Первый элемент внутреннего массива будет названием или id столбца, а второй значением ячейки.

Не пишите код, основываясь только на текущих входных данных. Нужно код писать так, чтобы он работал правильно с любыми входными данными нужного формата.

Пример №1.

Входные данные:

}

Выходные данные: first — First, second — Second

Пример № 2.

Входные данные:

first: 'First',
second: 'Second',
third: 'Third'

}

Выходные данные: first — First, second — Second, third — Third

ответ

const obj = {

first: "First",
second: "Second"

}

const keys = Object.keys(obj);

const values = Object.values(obj);

let result = '';

for (let i = 0; i ‹ keys.length; i += 1) {

if (i ‹ keys.length - 1) {
result += `${keys[i]} - ${values[i]}, `;
} else {
result += `${keys[i]} - ${values[i]}`;
}

}

console.log(result)
